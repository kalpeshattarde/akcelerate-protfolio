# 🎯 Quick Deploy: Prompt-Based Prototype Activation

Use this guide to instantly deploy any VelocityCore prototype with minimal configuration.

## One-Command Deploy (Choose Your Prototype)

### Project Management Prototype
```bash
npm run deploy:prototype -- --type project_management \
  --name "TaskFlow Pro" \
  --org "Your Company" \
  --platform railway/vercel/fly
```

## Prompt Workflow: From Concept to Live (5 Minutes)

### Step 1: Define Your Prototype (2 min)
```
I want to deploy [PROTOTYPE_TYPE] with these customizations:
- Business name: [YOUR_NAME]
- Target users: [USER_TYPE]  
- Key features: [FEATURES]
- Integrations needed: [INTEGRATIONS]
- Deployment preference: [PLATFORM]
- Domain name: [DOMAIN]
```

**Example:**
```
I want to deploy project_management with these customizations:
- Business name: TaskMaster Pro
- Target users: Software development teams
- Key features: Kanban boards, sprint planning, team collaboration
- Integrations needed: Slack, GitHub
- Deployment preference: Railway
- Domain name: app.taskmaster.pro
```

### Step 2: Generate Deployment Config (1 min)
```
Generate deployment config for my [PROTOTYPE] on [PLATFORM]:
- App name: [NAME]
- Database: PostgreSQL (managed)
- Storage: S3 or local
- Monitoring: Sentry + LogRocket
- Email: Resend
- Authentication: Google OAuth + email
```

### Step 3: Deploy & Configure (2 min)
```
Deploy my app:
1. Push to production on [PLATFORM]
2. Configure [DOMAIN] DNS
3. Create 3 test users: admin, user1, user2
4. Load 10 sample data records
5. Send me login credentials
6. Create monitoring alerts for:
   - Downtime > 5min
   - Error rate > 5%
   - Response time > 1s
```

## Environment Variables: Prompt-Generated

Automatically create `.env.production`:

```bash
# Generated by: deploy-prototype --env --type [TYPE]

# Core
NODE_ENV=production
DATABASE_URL=postgresql://user:pass@host:5432/db
REDIS_URL=redis://localhost:6379
JWT_SECRET=auto-generated-secret
PORT=8000

# Domain & CORS
DOMAIN=app.yourdomain.com
CORS_ORIGIN=https://app.yourdomain.com,https://yourdomain.com

# Stripe (Billing)
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Resend (Email)
RESEND_API_KEY=re_...
RESEND_FROM=noreply@yourdomain.com

# Twilio (SMS)
TWILIO_ACCOUNT_SID=ACxxxx
TWILIO_AUTH_TOKEN=xxxxxx

# Monitoring
SENTRY_DSN=https://xxx@sentry.io/xxx
LOG_LEVEL=info

# Feature Flags
ENABLE_SOCIAL_PUBLISHING=true
ENABLE_WEBHOOKS=true
ENABLE_API_KEYS=true

# Prototype-Specific
[PROTOTYPE_SPECIFIC_VARS]
```

## Fastest Deploy Path (10 Minutes Total)

### 1. Register Account (2 min)
```bash
# Go to your chosen platform
# - Railway: railway.app
# - Fly.io: fly.io
# - Vercel: vercel.com
# Choose GitHub, signup, authorize repo access
```

### 2. Connect Repository (1 min)
```bash
# In platform dashboard:
# New Project → Import from GitHub → Select velocitycore-prototypes
```

### 3. Add Environment Variables (2 min)
```bash
# Platform dashboard → Environment → Add:
DATABASE_URL=[provided by platform]
JWT_SECRET=[generate strong key]
NODE_ENV=production
CORS_ORIGIN=https://yourdomain.com

# Optional but recommended:
STRIPE_SECRET_KEY=sk_live_...
SENTRY_DSN=https://...
```

### 4. Deploy (3 min)
```bash
# Platform auto-deploys after each push
# OR manually trigger in dashboard
# Wait for build → done!
```

### 5. Configure Domain (2 min)
```bash
# In platform: Add Custom Domain
# In DNS provider: Add CNAME record pointing to platform
# Wait for SSL certificate (usually instant)
```

**Total: Ready to use!** 🚀

```bash

npm run deploy:version --app [APP_NAME] --version v1.0.5
```
